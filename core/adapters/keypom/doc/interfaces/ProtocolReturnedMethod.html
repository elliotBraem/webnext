Interface ProtocolReturnedMethod

Method information returned from the Protocol. This interface is exactly the same as the Method, except all the fields are snake cased instead of camel cased due to what the Protocol returns.

account_id_field?: string

Specifies what field Keypom should auto-inject the account that claimed the drop's ID into when calling the function. As an example, if the methodName was nft_mint and it expected a field receiver_id to be passed in, indicating who should receive the token, then the accountIdField would be receiver_id.

Optional attached_gas

attached_gas?: string

How much gas to attach to this method call. If none, all the gas is split between the parallel method calls in a given claim. If this is specified, the key can ONLY be used to call claim and no deposit_per_use can be specified. This leads the key to act like a method calling proxy instead of a linkdrop.
